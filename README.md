# ReversiAI

A fully playable Reversi (Othello) game with four AI agents of increasing sophistication, built from scratch in Python.

Designed to explore classical game tree search and modern deep learning approaches to game-playing AI.

## Overview:

This project implements a complete Reversi engine with a pygame GUI, supporting any combination of human and AI players.

The AI agents range from a simple random baseline to a convolutional neural network trained on self-play data.

The codebase is deliberately structured for clarity and extensibility - game logic, AI agents, and rendering are fully decoupled, making it easy to add new agents or swap components independently.

## Agents:

| Agent | Strategy |
|---|---|
| **Random** | Picks a uniformly random legal move. Baseline for evaluation. |
| **Greedy** | Maximizes pieces flipped in a single move. Fast and surprisingly competitive early-game. |
| **Minimax** | Alpha-beta pruning search with a blended positional heuristic (weighted board table + mobility + piece parity). Depth-configurable. |
| **Neural** | CNN trained on self-play data from mixed-strength agents. Evaluates board positions directly without explicit search. |

## Neural Network:

The neural agent uses a small convolutional network to evaluate board positions:

- **Input:** `(3, 8, 8)` tensor - separate channels for Black pieces, White pieces, and empty squares.
- **Architecture:** 3 convolutional blocks (64 -> 128 -> 128 filters) + 3 fully connected layers.
- **Output:** Scalar in `[-1, 1]` representing the positions value from Black's perspective.
- **Training:** MSE loss, Adam optimizer with learning rate scheduling, 80/20 training/validation split.

Training data is generated by having agents of varying strength play eachother (Random vs. Random, Minimax vs. Random, Greedy vs. Minimax, etc.), producing a diverse dataset of board positions labelled with game outcomes.

## Project structure:

```
ReversiAI/
├── agents/
│   ├── base_agent.py       # Abstract base class — enforces choose_move() interface
│   ├── human_agent.py      # UI-driven agent (click to move)
│   ├── random_agent.py     # Random baseline
│   ├── greedy_agent.py     # Greedy heuristic
│   ├── minimax_agent.py    # Minimax with alpha-beta pruning
│   └── neural_agent.py     # CNN-based evaluation agent
├── game/
│   ├── board.py            # Pure board data structure (no game logic)
│   ├── game.py             # Turn management, pass logic, win detection
│   └── constants.py        # All configuration in one place
├── ui/
│   ├── renderer.py         # pygame rendering (board, pieces, HUD)
│   └── menu.py             # Pre-game agent selection menu
├── scripts/
│   ├── generate_data.py    # Self-play data generation
│   └── train.py            # Neural network training pipeline
└── main.py                 # Entry point
```

## Getting Started:

**Install dependencies:**
```bash
pip install -r requirements.txt
```

**Run the game:**
```bash
python main.py
```

Select any combination of Human, Random, Greedy, Minimax, or Neural for each side from the menu.

**Training the neural agent:**
```bash
# Step 1 - generate self-play training data (warning: can take a long time):
python scripts/generate_data.py

# Step 2 - train the network (saves to models/reversi_net.pt):
python scripts/train.py
```

## Design Highlights:

- **Separation of concerns:** `Board` is a pure data structure with no knowledge of turns or win conditions. `Game` owns the state machine. Agents know nothing about rendering.
- **Uniform agent interface:** All agents (including human) implement the same `choose_move(board, valid_moves)` interface via an abstract base class, so the game loop never special-cases agent types.
- **Non-blocking AI:** AI moves are computed on a background thread so the pygame window stays responsive during search.
- **Positional heuristic:** The Minimax agent uses a weighted position table that heavily rewards corners and penalizes the cells adjacent to them, blended with mobility and piece parity scores that shift in weight as the game progresses.
